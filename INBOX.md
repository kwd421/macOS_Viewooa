# Viewooa Inbox

여기는 아직 정리되지 않은 생각을 편하게 적는 곳이다.

버그, 아이디어, 불편한 점, 갑자기 떠오른 제품 방향, 스크린샷 설명, 나중에 다시 볼 메모를 자연어로 적는다. 완벽하게 정리할 필요 없다. Obsidian에서 일기처럼 써도 된다.

에이전트는 이 파일을 먼저 훑고, 바로 실행할 일인지, 더 물어봐야 하는지, 아니면 정식 문서로 승격할 내용인지 판단한다. 방향이 합의된 내용만 아래 문서들로 옮긴다.

- `docs/product/ROADMAP.md`
- `docs/product/DESIGN.md`
- `docs/product/IMPLEMENTATION_MAP.md`
- `docs/product/FINDER_BEHAVIOR.md`
- `docs/product/KNOWN_ISSUES.md`
- `docs/engineering/AI_AGENT_RULES.md`

## 쓰는 법

그냥 편하게 쓴다. 짧게 써도 되고, 길게 써도 된다.

예시:

```md
## 2026-05-03

- 자체파인더: 처음 열 때 스크롤이 한 번 튄다.
- 자체파인더: 선택 색상이 기본 Finder랑 다르게 보인다.
- Viewooa: 사진 넘길 때 더 빠르게 느껴졌으면 좋겠다.
- 아이디어: Finder-style tabs는 나중에 넣고 싶지만, 구현 전에 다시 확인하기.
- 보류: 지금은 메타데이터 구조보다 실제 앱 사용감을 먼저 다듬기.
```

## 에이전트 처리 규칙

에이전트가 이 파일을 읽을 때는 다음 규칙을 따른다.

- 원본 메모는 사용자가 요청하지 않는 한 함부로 지우거나 정리하지 않는다.
- 모든 메모를 즉시 구현할 작업으로 취급하지 않는다.
- 각 메모를 대략 다음 상태로 분류한다: `Active`, `Done`, `Deferred`, `Cancelled`, `Needs Verification`, `Needs Clarification`.
- 제품 방향, 로드맵 상태, 아키텍처 경계, 실행 규칙이 바뀌는 내용만 정식 문서로 옮긴다.
- 같은 메모를 여러 문서에 중복으로 넣지 않는다. 각 문서에 들어갈 이유가 분명할 때만 나눈다.
- `DESIGN.md`, `IMPLEMENTATION_MAP.md`, `ROADMAP.md`와 충돌하면 조용히 고르지 말고 충돌을 사용자에게 알려준다.
- 사용자가 "전부 구현해"라고 해도, 보류된 큰 기능은 포함되는지 먼저 확인한다.
